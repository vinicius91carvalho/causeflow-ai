# QA Journal: WI-AC-019 (Clerk middleware: unauthenticated /dashboard redirects to /auth/sign-in)

## Date: 2026-07-09

## Result: PASS - implementation=true, integration=true, qa_passed

### Summary

AC-019 describes the middleware ensuring unauthenticated requests to protected routes
redirect to `/auth/sign-in`, while public routes like `/api/health/detailed` pass through.
The implementation on this branch uses local JWT auth (replacing Clerk per the
open-source-local-runtime context) but preserves the same behavioral contract.

### Independent Verification

1. **`GET /dashboard` without session → 307 redirect to `/auth/sign-in`** - VERIFIED
   - `curl -I http://localhost:3001/dashboard` → 307
   - Location: `/auth/sign-in?redirect_url=%2Fdashboard`
   - The `redirect_url` preserves the original URL for post-sign-in redirect

2. **Other non-public routes redirect identically** - VERIFIED
   - `/dashboard/analyses` → 307 → `/auth/sign-in?redirect_url=%2Fdashboard%2Fanalyses`
   - `/dashboard/billing` → 307 → `/auth/sign-in?redirect_url=%2Fdashboard%2Fbilling`
   - `/dashboard/integrations` → 307 → `/auth/sign-in?redirect_url=%2Fdashboard%2Fintegrations`

3. **`GET /api/health/detailed` returns 200 without session** - VERIFIED
   - Response: 200 OK, body `{"status":"degraded","version":"unknown","timestamp":"...","dependencies":[]}`
   - Graceful degradation when Core API is unavailable (expected outside docker-compose)

4. **All `/api/*` routes bypass auth** - VERIFIED
   - Middleware early-returns for `/api/` paths, delegating auth to `withAuth` server-side
   - `/api/billing/webhook` (404 in OSS build as Stripe is handled by Core) is not blocked by middleware

5. **Locale-prefixed public route `/auth/sign-in` is accessible** - VERIFIED
   - `/auth/sign-in` returns 200 (public route passes through to page handler)
   - `/en/auth/sign-in` correctly redirected to `/auth/sign-in` by intlMiddleware locale normalization

### Implementation notes

The middleware has been migrated from `clerkMiddleware()` to a local JWT cookie check
(per AC-046 / open-source-local-runtime). The `__session` httpOnly cookie replaces the
Clerk session; the JWT payload is decoded in the Edge Runtime (no signature verification —
deferred to `withAuth` which calls the Core's `/v1/auth/me`). The behavioral contract
of AC-019 is fully preserved.

### Verdict

All AC-019 steps pass independently against the running dev server on port 3001.
Zero defects found.
