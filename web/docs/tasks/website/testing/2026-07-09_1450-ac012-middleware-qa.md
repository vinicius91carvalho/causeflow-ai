## Defect found

**Spec says "45+ bot user-agent substrings" but implementation has only 20.**

AC-012 states: "Crawler detection: 45+ bot user-agent substrings (googlebot, bingbot, slurp, ...) never get redirected."

`apps/website/src/middleware.ts` defines `CRAWLER_UA_PATTERNS` with exactly 20 entries. The concrete Googlebot test passes, but the coverage breadth (20 vs 45+) does not meet spec.

## Evidence
- Server started on port 5173, dev mode `.env.local`
- Crawler: `curl -H 'User-Agent: Googlebot/2.1' http://127.0.0.1:5173/` -> HTTP 200
- Geo-redirect: `curl -H 'Cloudfront-Viewer-Country: BR' http://127.0.0.1:5173/` -> HTTP 307 Location /pt-br
- Accept-Language: `curl -H 'Accept-Language: pt-BR,en;q=0.8' http://127.0.0.1:5173/` -> HTTP 307 Location /pt-br
- Cookie: `curl -H 'Cookie: NEXT_LOCALE=pt-br' http://127.0.0.1:5173/` -> HTTP 307, Set-Cookie Max-Age=31536000
- Staging auth: restart with `NEXT_PUBLIC_DEPLOYMENT_STAGE=staging` -> no cookie -> 307 /staging-auth; valid cookie -> 200
- CRAWLER_UA_PATTERNS count: 20 (node -e confirms)
