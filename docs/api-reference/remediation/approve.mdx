description: "Approve a proposed remediation for execution."
---

## Overview

Approves a proposed remediation, making it eligible for execution. After approval, use [POST /v1/remediation/{remediationId}/execute](/api-reference/remediation/execute) to run it.

**Required role:** `admin`

<Warning>
  Only admins can approve remediations. This restriction exists because approved remediations make changes to production systems. Ensure you have reviewed the remediation steps before approving.
</Warning>

## Path parameters

<ParamField path="remediationId" type="string" required>
  The unique identifier of the remediation to approve. Example: `rem_01HX9VTPQR3KF8MZWBYD5N6JCE`.
</ParamField>

## Response

Returns `200 OK` with the updated remediation status.

```json
{
  "remediationId": "rem_01HX9VTPQR3KF8MZWBYD5N6JCE",
  "status": "approved"
}
```

| Field           | Type   | Description                                  |
| --------------- | ------ | -------------------------------------------- |
| `remediationId` | string | The remediation that was approved             |
| `status`        | string | Always `"approved"` after a successful call  |

Returns `404 Not Found` if the remediation does not exist.
Returns `403 Forbidden` if the authenticated user is not an `admin`.
Returns `409 Conflict` if the remediation has already been approved, rejected, or executed.

## Examples

<CodeGroup>

```bash curl
curl https://api.causeflow.ai/v1/remediation/rem_01HX9VTPQR3KF8MZWBYD5N6JCE/approve \
  -X POST \
  -H "Authorization: Bearer <token>"
```

```typescript TypeScript
const remediationId = "rem_01HX9VTPQR3KF8MZWBYD5N6JCE";

const response = await fetch(
  `https://api.causeflow.ai/v1/remediation/${remediationId}/approve`,
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${token}`,
      "Content-Type": "application/json",
    },
  }
);

if (response.status === 403) {
  throw new Error("Only admins can approve remediations.");
}

const result = await response.json();
console.log(`Remediation ${result.remediationId} is now: ${result.status}`);
```

</CodeGroup>
