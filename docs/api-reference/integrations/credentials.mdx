title: "Store credentials"
api: "POST /v1/integrations/{provider}/credentials"
description: "Store API credentials for a provider that uses API key authentication."
---

## Overview

Saves API credentials for integrations that use API key authentication (not OAuth). Credentials are encrypted at rest and never returned in plaintext after this request.

**Required role:** `admin`

## Request headers

<ParamField header="Authorization" type="string" required>
  JWT Bearer token. Format: `Bearer <your-jwt>`.
</ParamField>

## Path parameters

<ParamField path="provider" type="string" required>
  The integration provider. Example: `datadog`. Must be a provider with `authType: api_key` in the catalog.
</ParamField>

## Request body

<ParamField body="credentials" type="object" required>
  Key-value map of credential fields. The required keys vary by provider — see the [integration catalog](/api-reference/integrations/catalog) for `requiredFields` per provider. Example for Datadog: `{ "api_key": "...", "application_key": "..." }`.
</ParamField>

<ParamField body="label" type="string">
  Optional human-readable label to identify this credential set. Useful if you have multiple environments.
</ParamField>

## Response

Returns `201 Created`.

```json
{
  "credentialId": "crd_01HX9VTPQR3KF8MZWBYD5N6JCE",
  "provider": "datadog",
  "label": "Production",
  "status": "validated",
  "createdAt": "2024-04-01T10:00:00Z"
}
```

| Field | Type | Description |
| --- | --- | --- |
| `credentialId` | string | Unique credential identifier |
| `provider` | string | Provider name |
| `label` | string | Optional label you provided |
| `status` | string | `validated` if credentials were successfully tested, `unverified` otherwise |
| `createdAt` | string | ISO 8601 timestamp |

### Error responses

| Status | Error code | Description |
| --- | --- | --- |
| `400` | `validation_error` | Missing required credential fields |
| `401` | `unauthorized` | Missing or invalid JWT |
| `403` | `forbidden` | Caller lacks `admin` role |
| `422` | `credential_validation_failed` | CauseFlow tested the credentials against the provider and they were rejected |

## Examples

<CodeGroup>

```bash curl
curl https://api.causeflow.ai/v1/integrations/datadog/credentials \
  -X POST \
  -H "Authorization: Bearer eyJhbGc..." \
  -H "Content-Type: application/json" \
  -d '{
    "credentials": {
      "api_key": "dd_api_key_EXAMPLE",
      "application_key": "dd_app_key_EXAMPLE"
    },
    "label": "Production"
  }'
```

```typescript TypeScript
const response = await fetch(
  "https://api.causeflow.ai/v1/integrations/datadog/credentials",
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${token}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      credentials: {
        api_key: process.env.DD_API_KEY!,
        application_key: process.env.DD_APP_KEY!,
      },
      label: "Production",
    }),
  }
);

if (response.status === 422) {
  console.error("Credentials rejected by Datadog — check your keys");
}

const cred = await response.json();
console.log("Status:", cred.status);
```
