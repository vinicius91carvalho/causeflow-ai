title: "Start investigation"

description: "Start a multi-agent investigation for an incident."
---

## Overview

Dispatches a team of specialist AI agents to investigate an incident. Each agent analyzes the incident from a different angle (infrastructure, code changes, dependencies, etc.) and contributes evidence to the root cause analysis.

Starting an investigation consumes **1 investigation** from your monthly plan quota.

**Required role:** `admin` or `member`

## Path parameters

<ParamField path="incidentId" type="string" required>
  The unique identifier of the incident to investigate. Example: `inc_01HX9VTPQR3KF8MZWBYD5N6JCE`.
</ParamField>

## Response

Returns `200 OK` when the investigation has been started successfully.

```json
{
  "incidentId": "inc_01HX9VTPQR3KF8MZWBYD5N6JCE",
  "status": "investigating",
  "agentsDispatched": [
    "database-specialist",
    "performance-analyzer",
    "deployment-tracker"
  ],
  "message": "Investigation started. Results will be available once agents complete their analysis."
}
```

| Field              | Type     | Description                                                              |
| ------------------ | -------- | ------------------------------------------------------------------------ |
| `incidentId`       | string   | The incident being investigated                                          |
| `status`           | string   | Always `"investigating"` after a successful dispatch                     |
| `agentsDispatched` | string[] | The specialist agents that have been dispatched for this investigation   |
| `message`          | string   | Human-readable confirmation message                                      |

Investigation results are available asynchronously. Poll [GET /v1/investigation/{incidentId}](/api-reference/investigation/get-results) or listen for the `investigation.completed` webhook event to know when results are ready.

<Note>
  Starting an investigation consumes 1 investigation from your monthly quota. To check your remaining allowance, see the `investigationsRemaining` field in the [Get incident](/api-reference/incidents/get-incident) response.
</Note>

## Examples

<CodeGroup>

```bash curl
curl https://api.causeflow.ai/v1/investigation/inc_01HX9VTPQR3KF8MZWBYD5N6JCE \
  -X POST \
  -H "Authorization: Bearer <token>"
```

```typescript TypeScript
const incidentId = "inc_01HX9VTPQR3KF8MZWBYD5N6JCE";

const response = await fetch(
  `https://api.causeflow.ai/v1/investigation/${incidentId}`,
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${token}`,
      "Content-Type": "application/json",
    },
  }
);

const result = await response.json();

console.log(`Status: ${result.status}`);
console.log(`Agents dispatched: ${result.agentsDispatched.join(", ")}`);
```

</CodeGroup>
