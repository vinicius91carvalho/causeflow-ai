---
title: "Execute remediation"
api: "POST /v1/remediation/{remediationId}/execute"
description: "Execute an approved remediation."
---

## Overview

Executes a previously approved remediation. CauseFlow runs each step in order and tracks progress in real time. The remediation must be in `approved` status before execution can begin.

**Required role:** `admin`

<Warning>
  Executing a remediation makes changes to your production systems. Verify that the remediation has been reviewed and that you are executing it during an appropriate maintenance window if required.
</Warning>

## Path parameters

<ParamField path="remediationId" type="string" required>
  The unique identifier of the remediation to execute. Must be in `approved` status. Example: `rem_01HX9VTPQR3KF8MZWBYD5N6JCE`.
</ParamField>

## Response

Returns `200 OK` when execution has been initiated successfully.

```json
{
  "remediationId": "rem_01HX9VTPQR3KF8MZWBYD5N6JCE",
  "status": "executing"
}
```

| Field           | Type   | Description                                                             |
| --------------- | ------ | ----------------------------------------------------------------------- |
| `remediationId` | string | The remediation being executed                                          |
| `status`        | string | Always `"executing"` immediately after a successful execution request   |

Execution is asynchronous. Poll [GET /v1/remediation/{incidentId}](/api-reference/remediation/list-remediations) to monitor step-level progress, or listen for the `remediation.completed` and `remediation.failed` webhook events.

### Status transitions after execution

```
approved → executing → completed
                     → failed
```

### Error responses

| Status | Error code                  | Description                                                     |
| ------ | --------------------------- | --------------------------------------------------------------- |
| `404`  | `remediation_not_found`     | No remediation exists with the given ID                         |
| `403`  | `forbidden`                 | The authenticated user is not an `admin`             |
| `409`  | `invalid_status`            | The remediation is not in `approved` status                     |

<Note>
  A remediation must be in `approved` status before execution. Call [POST /v1/remediation/{remediationId}/approve](/api-reference/remediation/approve) first if it is still awaiting approval.
</Note>

## Examples

<CodeGroup>

```bash curl
curl https://api.causeflow.ai/v1/remediation/rem_01HX9VTPQR3KF8MZWBYD5N6JCE/execute \
  -X POST \
  -H "Authorization: Bearer <token>"
```

```typescript TypeScript
const remediationId = "rem_01HX9VTPQR3KF8MZWBYD5N6JCE";

const response = await fetch(
  `https://api.causeflow.ai/v1/remediation/${remediationId}/execute`,
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${token}`,
      "Content-Type": "application/json",
    },
  }
);

if (response.status === 409) {
  throw new Error(
    "Remediation is not in approved status. Approve it before executing."
  );
}

const result = await response.json();
console.log(`Remediation ${result.remediationId} is now: ${result.status}`);
// Poll GET /v1/remediation/{incidentId} to monitor step progress
```

</CodeGroup>
