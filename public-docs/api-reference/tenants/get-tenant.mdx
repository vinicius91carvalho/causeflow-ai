---
title: "Get tenant"
api: "GET /v1/tenants/{tenantId}"
description: "Get details of a specific tenant."
---

## Overview

Returns the full details of a specific tenant, including its settings and current status.

**Required role:** Any authenticated user from the same tenant

## Path parameters

<ParamField path="tenantId" type="string" required>
  The unique identifier of the tenant to retrieve. Example: `ten_EXAMPLE_01HX9VTPQR3KF8MZ`.
</ParamField>

## Response

Returns `200 OK` with the full tenant object.

```json
{
  "tenantId": "ten_EXAMPLE_01HX9VTPQR3KF8MZ",
  "name": "Acme Corp",
  "slug": "acme-corp",
  "status": "active",
  "plan": "community",
  "createdAt": "2026-01-15T09:30:00Z",
  "settings": {
    "awsRoleArn": "<your-aws-iam-role-arn>",
    "awsExternalId": "causeflow-ext-id-acme",
    "awsRegion": "us-east-1",
    "autoRemediation": false
  }
}
```

| Field                      | Type    | Description                                                         |
| -------------------------- | ------- | ------------------------------------------------------------------- |
| `tenantId`                 | string  | Unique identifier for the tenant                                    |
| `name`                     | string  | The tenant display name                                             |
| `slug`                     | string  | The URL-safe tenant identifier                                      |
| `status`                   | string  | Tenant status: `active` or `suspended`                              |
| `plan`                     | string  | Deployment label (`community` in OSS; not a paid tier)               |
| `createdAt`                | string  | ISO 8601 timestamp of when the tenant was created                   |
| `settings.awsRoleArn`      | string  | ARN of the AWS IAM role used for AWS integrations                   |
| `settings.awsExternalId`   | string  | External ID used when assuming the AWS IAM role                     |
| `settings.awsRegion`       | string  | Primary AWS region for this tenant's infrastructure                 |
| `settings.autoRemediation` | boolean | Whether automatic remediation execution is enabled                  |

### Error responses

| Status | Error code    | Description                                            |
| ------ | ------------- | ------------------------------------------------------ |
| `403`  | `forbidden`   | The authenticated user does not belong to this tenant  |
| `404`  | `not_found`   | No tenant exists with the provided `tenantId`          |

## Examples

<CodeGroup>

```bash curl
curl https://api.causeflow.ai/v1/tenants/ten_EXAMPLE_01HX9VTPQR3KF8MZ \
  -H "Authorization: Bearer <token>"
```

```typescript TypeScript
const tenantId = "ten_EXAMPLE_01HX9VTPQR3KF8MZ";

const response = await fetch(
  `https://api.causeflow.ai/v1/tenants/${tenantId}`,
  {
    headers: {
      Authorization: `Bearer ${token}`,
      "Content-Type": "application/json",
    },
  }
);

if (response.status === 404) {
  throw new Error("Tenant not found.");
}

const tenant = await response.json();
console.log(`Tenant: ${tenant.name} (${tenant.plan} plan)`);
```

</CodeGroup>
